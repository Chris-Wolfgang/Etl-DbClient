Remarks

The caller owns the DbConnection lifetime — the extractor does not open, close, or dispose it. The connection must be open before calling ExtractAsync.

An optional DbTransaction can be provided for isolation level control. The extractor never commits or rolls back the transaction.

Command timeout uses the Dapper/ADO.NET default (typically 30 seconds). A dedicated CommandTimeout property is planned (see GitHub issue #25).

The default total count implementation. Wraps CommandText in SELECT COUNT(*) FROM (...) AS _count and executes it using the same connection, parameters, and transaction as the extraction query. Assign this to TotalCountQuery to enable the built-in behavior.

Remarks

Trailing semicolons are stripped automatically. If the command text contains an ORDER BY clause, some database providers (e.g. SQL Server) may reject it inside a derived table. Use a custom TotalCountQuery in that case.

TotalCountQuery

When non-null, this function is invoked before extraction begins to determine the total record count, which is then reported via TotalItemCount. Assign DefaultTotalCountQuery to use the library's built-in SELECT COUNT(*) subquery, or supply a custom function for a more efficient query. Defaults to null (total count is not fetched).
